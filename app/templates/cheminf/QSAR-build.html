QSAR Builder

Quantitative Structure-Activity Relationship (QSAR) models relate chemical structures to observed biological activities. A model is defined by a feature set (e.g., ECFP6) and a learning algorithm (e.g., Random Forest). Select a dataset, feature(s), and algorithm(s), then click Build QSAR. Saved models are available in Cheminformatics → QSAR Predictor.

Select dataset(s):
You may select one or more datasets.
Select features:
ECFP6 encodes circular substructural patterns around each atom and is widely used for QSAR classification and regression.
FCFP6 is similar to ECFP6 but emphasizes functional or pharmacophoric features instead of only atom identity.
RDKit Descriptors are calculated molecular properties such as physicochemical and topological features.
You may select one or more feature sets to compare model performance.
Select algorithm:
Random Forest (RF) is an ensemble method that combines many decision trees and is often a strong default choice.
k-Nearest Neighbors (kNN) predicts a compound based on the most similar compounds in the training set.
Support Vector Machine (SVM) learns an optimal boundary between classes or fits relationships in a high-dimensional space.
Multiple algorithms can be selected to build and compare models.
Select activity type:
Classification is used when activity values represent categories such as active/inactive or toxic/non-toxic.
Regression is used when activity values are continuous numerical measurements such as IC50, EC50, or other assay values.
Classification models assign class labels, while regression models predict numeric values.
